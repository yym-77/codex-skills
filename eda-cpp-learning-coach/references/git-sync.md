# Git 提交与同步

## 动作语义

- **结束今天**：整理既有草稿、当前状态、复习条目和派生索引；不自动执行版本提交或远端同步。
- **提交记录**：检查候选文件、脱敏结果和待提交差异后执行本地提交；不执行远端同步。
- **同步记录**：检查目标仓库、分支和待同步提交后再执行同步。
- **结束今天并同步记录**：用户已授权依次整理、检查、提交和同步；任何安全门禁失败都必须停止。

执行前读取成长仓库中的 `state/config.yaml`，并确认目标位于 `allowed_remotes` 中。空列表表示禁止远端同步。

## 允许同步的成长数据

私有成长仓库可以同步以下已检查内容：

- `state/*.yaml`：当前主题、能力进度、复习队列、公开源码注册和配置；
- `records/**/*.md`：权威技术记录；
- `decisions/**/*.md`：架构决策；
- `planning/*.md`：学习积压；
- `indexes/*.md`：可重建索引；
- `profile.md` 与 `skill-source.yaml`；
- `experiments/` 中用户自行编写的人工最小实验。

不得同步：

- `local-only/`；
- 真实工作源码、原始日志、截图、转储、工程文件和专有数据；
- 机器专属路径与本地源码目录；
- 无法确认来源和授权的附件。

## 提交前检查

1. 确认仓库仍为预期的私有成长仓库。
2. 检查分支和工作区，不混入无关改动。
3. 只选择本轮明确更新的文件。
4. Markdown 权威记录执行自动扫描和人工保密复核；YAML 状态执行人工字段检查。
5. 展示差异摘要，检查标题、路径、标签和提交说明。
6. 使用简短、抽象的提交说明，不包含真实工作标识。

## 同步前检查

1. 再次确认目标、分支和待同步提交。
2. 获取远端状态，只允许不会改写既有历史的安全更新。
3. 发生分叉、冲突、可见性变化或认证异常时停止并报告。

禁止自动改写已发布历史、自动解决内容冲突、切换仓库可见性或提交无关改动。
