# Git 提交与同步

## 动作语义

- **结束今天**：只整理既有本地草稿、活动主题状态、既有复习条目和派生索引；不创建新技术记录，不升级深度复盘，也不执行版本提交或远端同步。
- **提交记录**：检查候选文件、脱敏结果和待提交差异后执行本地提交；不执行远端同步。
- **同步记录**：检查目标仓库、分支和待同步提交后再执行同步。
- **结束今天并同步记录**：用户已授权依次整理、检查、提交和同步；任何安全门禁失败都必须停止。

执行前读取用户数据目录中的 `state/config.yaml`。配置不存在时，只有获得本地写入授权后才能从模板初始化，并确认 `data_root` 和 IANA `timezone`。

## Remote 允许名单

- 使用 `allowed_remotes` 字符串列表保存明确允许的目标。
- 空列表表示禁止远端同步。
- 比较时可以规范化同一仓库的不同 URL 表达，但必须同时核对主机、所有者和仓库路径。

## 提交前检查

1. 确认当前仓库和目标位于允许名单中，并确认目标仍为预期的私有成长仓库。
2. 检查分支和工作区；发现无关改动时不覆盖、不顺手提交。
3. 首版只允许同步明确点名的 `records/**/*.md` 与 `decisions/**/*.md` 权威记录。
4. 只暂存本轮明确生成或更新的文件，禁止默认包含整个工作区。
5. 对每个候选记录运行敏感信息扫描和同步元数据门禁，并进行人工保密复核。
6. 展示待提交差异摘要，检查标题、路径、标签和提交信息是否泄露真实标识。
7. 使用简短、抽象的提交说明，不包含客户、项目、内部类名或故障细节。

## 同步前检查

1. 再次确认目标、分支和待同步提交。
2. 获取远端状态，只允许不会改写既有历史的安全更新。
3. 发生分叉、冲突、仓库可见性变化或认证异常时停止并报告。

禁止自动改写已发布历史、自动解决内容冲突、切换仓库可见性或提交无关改动。

若敏感信息已经进入版本历史，普通删除文件不能完成清除。停止同步，先处理凭据和历史清理问题。
