# 真实问题分析

## 原则

优先恢复正确行为和验证结果，不把紧急问题变成强制学习作业。只有用户明确要求时才持久化；解决问题后再判断是否值得形成记录。

## 输入边界

- 先确认现象、复现条件、已有证据、近期变化、期望结果和验收方式。
- 不主动扫描真实工作仓库。用户可以主动提供自己有权处理的有限、已脱敏代码片段、截图、接口契约、人工最小复现或抽象对象关系。
- 用户明确授权分析工作材料时，只处理点名的最小只读范围，不扫描整个仓库，不修改或提交源仓库。
- 对话中可以围绕授权片段定位问题；任何成长记录只能保存重新抽象的方法、人工实验和结论，不能保存原始工作材料。

## 分析流程

1. 用一句话定义问题：对象、触发条件、异常和影响。
2. 列出已验证事实与缺失信息，不在事实区写根因猜测。
3. 按概率、区分度和验证成本排列假设。
4. 为每个假设给出可观察结果和最小验证动作。
5. 执行或建议日志点、断言、最小复现、测试或对照实验。
6. 只有证据闭合后才写最终根因；否则保持“高概率假设”状态。
7. 给出最小修复、备选方案、代价、回滚范围和回归检查。
8. 提炼错误模式、第一个验证动作、适用边界和预防规则。

## 表达规范

| 类别 | 要求 |
|---|---|
| 已验证事实 | 由源码、日志、测试或运行结果支持 |
| 高概率假设 | 写明依据和下一步验证方式 |
| 待排除可能 | 放低优先级，不抢占主线 |
| 最终结论 | 写明触发条件、根因、修复和验证结果 |

## 记录价值判断

- 一次性、调用明确、无迁移价值：仅在当日摘要中简记或不记录。
- 形成明确规律、排查方法或预防规则：建议快速记录。
- 高风险或高影响：建议深度复盘。
- 非高风险问题满足“重复出现、多个合理假设、形成可复用方法、具有迁移价值”中任意两项：建议深度复盘。

只建议升级，最终写入和升级由用户明确触发。
