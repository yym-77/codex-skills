# 主题学习与源码取证

## 目标

围绕一个能力问题建立可验证的认知模型。源码阅读是取证方式，不是完成数量目标；不要求按周轮换组件，也不强求每个结论迁移到项目。

## 源码定位顺序

1. 读取成长记录仓库中的 `state/source-registry.yaml`。
2. 若条目引用 `local-only/source-paths.yaml`，只读取当前机器登记的本地路径，不把机器路径写回公共 Skill 或同步记录。
3. 优先使用已登记的本地只读源码；没有本地源码时，再使用明确允许访问的公开远程仓库。
4. 读取前确认仓库、模块、分支或标签、提交、编译器和平台。
5. 未固定版本时只能给出通用方向，不能把某个实现细节写成稳定结论。

## 开始前

1. 确认能力问题、当前表现、期望表现和本次时间预算。
2. 选择 10 分钟、30～45 分钟或 60～90 分钟强度。
3. 只选择一条调用链或一个机制，旁支进入积压清单。
4. 第三方源码默认只读；人工实验放在成长记录仓库的 `experiments/`。

## 学习闭环

1. **预测**：让用户闭卷预测行为、调用路径或关键不变量，并说明依据。用户要求直接解释时跳过强制预测。
2. **定位**：从公开接口进入关键定义、数据成员和控制分支，不从内部符号反向堆砌细节。
3. **分层**：分别标注标准规定、具体实现、实验观察和推测。
4. **解释**：提炼生命周期、所有权、异常安全、复杂度、并发或性能上的设计权衡。
5. **实验**：用人工构造的最小程序验证一个关键判断，记录预期、构建环境、实际结果和反例。
6. **修正**：比较初始预测与证据，说明认知模型发生了什么变化。
7. **变式**：改变一个条件，例如对象类型、容量、异常路径、线程或上下文，检查结论是否仍成立。
8. **迁移**：先验证共享不变量，再讨论项目应用；允许结论为“当前场景不适用”。
9. **自检**：要求用户脱离源码讲清调用链、不变量、边界和第一个验证动作。

## 最小实验记录

- 研究问题与预测
- 编译器、标准版本、平台和构建命令
- 最小代码与输入
- 预期与实际结果
- 能支持什么结论、不能支持什么结论
- 反例或失效边界

优先使用用户自写的独立代码，不依赖公司工程、专有数据或未公开接口。

## 降级与停止

- 遇到生成代码、宏分支、平台扩展或缺失依赖时，停在可验证边界并列出缺失信息，不编造调用链。
- 内部实现随版本变化时，记录来源和适用范围，不把某个实现写成语言标准。
- 旁支超过当前问题范围时先在对话中列为候选；只有用户明确要求维护积压清单时，才写入积压事实源。
