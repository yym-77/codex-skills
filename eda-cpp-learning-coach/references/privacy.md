# 保密、脱敏与安全停止条件

## 来源分级

| 来源 | 默认读取 | 可写入成长记录 |
|---|---|---|
| 公开标准库、官方文档和开源实现 | 可以 | 记录公开链接、版本和必要短片段 |
| 用户自行编写的实验代码 | 可以 | 可以，仍检查密钥和个人路径 |
| 公司、客户或真实工作材料 | 不主动读取；经用户确认有权处理后，仅处理点名的最小范围 | 禁止原文；仅允许用户确认后的重新抽象方法、人工实验和结论 |

用户确实要求分析工作材料时：

1. 要求用户确认拥有处理权限，并指定有限文件、截图、脱敏片段或抽象对象关系。
2. 只读最小范围，不扫描整个仓库，不修改或提交工作源码。
3. 可以在当前对话中分析用户主动提供的有限内容，但不得持久化真实符号名、路径、调用栈、内部版本、URL、参数、截图、日志和数据。
4. 成长记录从零编写通用伪代码、人工实验或抽象机制，不复制工作代码片段。
5. 无法确认授权、范围或脱敏质量时立即停止。

## 写入前检查

- 将公司、客户、产品、项目、命名空间、类名和业务字段改为通用抽象名。
- 删除或抽象绝对路径、机器名、用户名、内部服务器、仓库地址和私网 IP。
- 删除密钥、Token、密码、求解器参数、专有格式、完整日志、截图、转储和工程文件。
- 检查标题、slug、标签、链接、代码块、附件名和 Git 提交信息。
- `Private` 仓库不是外传许可，脱敏通过也不等于获得发布授权。

## 自动扫描

只对准备写入或提交的候选 Markdown 文件运行：

```text
python "SKILL_DIR/scripts/sensitive_scan.py" FILE.md [FILE2.md ...]
```

扫描器不得遍历未指定目录、联网、读取链接附件、修改源文件或自动脱敏。命中阻断项时停止写入或提交；只有人工复核后才能设置 `redaction_checked: true` 和 `sync_allowed: true`。

准备 commit 或 push 时必须增加同步元数据门禁：

```text
python "SKILL_DIR/scripts/sensitive_scan.py" --require-sync-metadata FILE.md [FILE2.md ...]
```

## 安全停止条件

出现以下任一情况立即停止持久化或同步：

- 无法确认资料是否允许被当前工具处理。
- 内容仍含真实工作源码、客户数据、密钥、内部 URL、路径、日志、截图、转储或专有文件。
- 抽象后仍可从唯一版本、架构组合或业务语义反推出真实项目。
- 目标仓库不是配置允许的私有成长仓库。
- 用户只授权记录，但即将发生 commit、push、删除或历史改写。
- 检测到并发改动、远端分叉或无法解释的无关文件。
